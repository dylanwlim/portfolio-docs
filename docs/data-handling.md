# Data Handling

## Public Documentation Boundary

The portfolio site is a public information surface. Do not send secrets, private customer data, or confidential implementation details through public channels linked from the site.

## Do Not Post

- Secrets, tokens, API keys, passwords, private certificates, or environment variables.
- Internal URLs, private routes, unpublished architecture, or proprietary workflows.
- Customer, client, patient, student, account, payment, booking, transcript, portfolio, or personal data.
- Screenshots that reveal private dashboards, private repositories, personal records, or non-public business context.

## Source Code

Production source code remains private unless a repository is explicitly linked as public. Public docs should stay focused on use, status, privacy, support, and safe product scope.
